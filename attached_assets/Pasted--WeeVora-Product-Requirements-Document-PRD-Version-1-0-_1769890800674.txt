# WeeVora Product Requirements Document (PRD)
**Version:** 1.0  
**Last Updated:** January 31, 2026  
**Product:** WeeVora - Lake County Summer Camp Directory  
**Tagline:** "Never miss registration again"
---
## 1. Executive Summary
WeeVora is a comprehensive summer camp discovery and planning platform for families in Lake County, Illinois. The platform aggregates camp information from multiple providers into a single, searchable directory with real-time registration status tracking, helping parents navigate the complex landscape of summer programming without missing critical deadlines.
### Core Value Proposition
- **For Parents:** Eliminates the need to track dozens of websites, remembering registration dates, and manually comparing options
- **Differentiation:** Registration-first approach with proactive status tracking vs. static camp listings

## 5. Feature Specifications
### 5.1 Homepage
**Hero Section:**
- Full-width with background image (`hero-playground.png` at 30% opacity)
- Decorative organic blob shapes (subtle, brand colors)
- Headline: "Never miss registration again"
- Subhead: "Your copilot for summer camps, activities, and keeping the kids busy."
- CTAs: "Find Camps Near You" (primary), "How It Works" (secondary)
- No post-it notes or floating cards (removed for cleaner hero image display)
**Features Section:**
- 3-column grid on desktop, stacked on mobile
- Paper-craft icon illustrations (20px border radius)
- Features:
  1. Camp Directory - "Every summer camp in Lake County with registration dates, costs, and details."
  2. Smart Search & Filters - "Find exactly what you need by age, location, schedule, and activity type."
  3. Always Up-to-Date - "We keep camp information current so you don't have to hunt across dozens of websites."
**Stats Section:**
- "50+ Camps Listed"
- "100% Free to Use"
### 5.2 Camp Directory (`/camps`)
**Layout:**
- Sidebar filters (desktop) / Bottom sheet filters (mobile)
- Grid of camp cards (responsive: 1/2/3 columns)
- Results count header
**Filter Options:**
| Filter | Type | Options |
|--------|------|---------|
| Search | Text input | Searches name, organization, description |
| Categories | Multi-select checkboxes | All camp categories |
| Age | Range slider | Min/max age filter |
| Location | Multi-select | Cities in Lake County |
| Price | Range slider | Min/max price |
| Registration Status | Toggle buttons | All, Open Now, Coming Soon |
| Extended Hours | Checkbox | Only show camps with extended hours |
**Mobile Filter Sheet:**
- Bottom sheet (85vh height)
- Rounded top corners (24px)
- Scroll behavior: `overscrollBehavior: 'contain'` (prevents jump to top on selection)
- Search input with proper padding to prevent focus ring clipping
**Sorting:**
- Default: Registration date (soonest first)
- Options: Name A-Z, Name Z-A, Registration Date
- **Critical:** Camps without registration details always sorted to bottom
### 5.3 Camp Cards
**Information Displayed:**
- Camp name
- Organization
- Category badges (color-coded)
- Age range
- Location (city)
- Registration status (see 5.4)
- "View Details" link
**Cards Without Registration Dates:**
- Display "Registration details not yet available" in muted italics
- No status indicator shown
- Visual deprioritization: 60% opacity
- Sorted to bottom of all listings
**Hover State:**
- Lift effect (translateY -8px)
- Enhanced shadow
### 5.4 Registration Status Logic
**Priority Order (highest to lowest):**
1. **Closed** (if `registration_closes` date has passed)
   - Red dot indicator
   - Text: "Registration closed"
2. **Upcoming** (if `registration_opens` is in the future)
   - Yellow dot indicator
   - Text: "Registration opens [date]"
3. **Waitlist Only** (if `waitlist_only` checkbox is true)
   - Red dot indicator
   - Text: "Waitlist only"
4. **Open** (if registration is currently open)
   - Green dot indicator
   - Text: "Registration closes [date]"
5. **No Data** (if both dates are missing)
   - No status indicator
   - Text: "Registration details not yet available"
   - Muted, italic styling
### 5.5 Camp Detail Page (`/camps/:slug`)
**Layout:**
- Breadcrumb navigation
- Hero section with camp name, organization, key details
- Tabbed content or sections for:
  - Overview/Description
  - Schedule & Dates
  - Pricing
  - Location
  - Additional Information
**Similar Camps Section:**
- "You might also like" recommendations
- Filter by same category or age range
- **Critical:** Camps with full registration details prioritized over incomplete entries
**SEO:**
- Dynamic Schema.org Event markup
- Meta tags from camp data
### 5.6 Contact Form
**Fields:**
- Name (required)
- Email (required)
- Subject (optional)
- Message (required)
**Backend:** Supabase Edge Function (`submit-contact`)

- Canonical URLs on all pages
- Responsive viewport meta tag
- robots.txt configuration
### 7.2 Structured Data
Implemented via `SchemaMarkup.tsx`:
| Schema Type | Page |
|-------------|------|
| Organization | All pages |
| WebSite | Homepage |
| ItemList | Camp directory |
| Event | Camp detail pages |
| BreadcrumbList | Camp detail pages |
### 7.3 Content Guidelines
- Title tags: Under 60 characters, include main keyword
- Meta descriptions: Under 160 characters
- Single H1 per page matching page intent
- Semantic HTML (`<header>`, `<main>`, `<section>`, `<footer>`)
- Descriptive alt text on all images
- Lazy loading for images
